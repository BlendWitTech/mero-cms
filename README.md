# Mero CMS

**Mero CMS** is a modular, self-hosted content management system built by [Blendwit Tech](https://blendwit.com). It lets you enable only the features your project needs — blogs, portfolio, team, services, testimonials, SEO, analytics, and more — then deploy a theme that automatically seeds its own content.

## What It Does

- **Modular backend** — enable/disable content modules without touching code
- **Setup wizard** — first-run wizard creates the DB schema, admin account, and seeds demo content
- **Theme system** — upload ZIP themes; each theme declares its required modules and seeds its own content
- **Role-based access control** — Super Admin, Admin, and custom roles with per-permission granularity
- **Public API** — themes fetch data from a single `/public/site-data` endpoint
- **SEO toolkit** — per-page meta, sitemap, robots.txt, and redirects
- **Analytics** — Google Analytics 4 dashboard embedded in the admin UI
- **Media manager** — upload and manage images/files used across content

## Theme System

Themes are standalone Next.js apps that call the CMS public API. They are self-contained — each theme declares what modules it needs and provides its own seed data.

**How a theme works:**
1. Theme has a `theme.json` with metadata, required modules, and seed data
2. Backend discovers themes in the `themes/` directory automatically
3. Admin clicks **Setup** → backend seeds the theme's menus, posts, testimonials, etc.
4. Admin clicks **Activate** → backend marks it as the active theme
5. Theme app fetches all its data from `GET /public/site-data`

**Packaging a custom theme for upload:**
```bash
node scripts/zip-theme.js <theme-slug>
# Output: themes/<theme-slug>.zip
# Then: Admin → Appearance → Themes → Upload Theme
```

## Quick Start (Local)

```bash
# 1. Clone (requires license or contributor access)
git clone https://github.com/blendwit-tech/mero-cms.git
cd mero-cms

# 2. Backend
cd backend
cp .env.development.example .env
# Edit .env — set DATABASE_URL and JWT_SECRET
npm install
npx prisma migrate dev
npm run start:dev

# 3. Frontend (new terminal from project root)
cd frontend
cp .env.development.example .env.local
npm install
npm run dev

# 4. Open http://localhost:3000/setup
```

For Railway + Vercel deployment, see [SETUP.md](SETUP.md).
For coding patterns and architecture, see [DEVELOPER_GUIDE.md](DEVELOPER_GUIDE.md).

## Scripts

From the repo root:

```bash
npm run dev              # Start backend + admin UI (development)
npm run dev:all          # Start backend + admin UI + a theme
npm run build            # Build backend and frontend for production
npm run zip-theme <slug> # Package a theme directory into a ZIP
npm run db:init          # Run Prisma migrations
npm run db:seed          # Seed database with default data
```
